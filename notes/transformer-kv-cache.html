训练出来的权重矩阵，是模型的"知识"，固定不变。

三个容易混淆的关键点，以及 Unembedding 层

Decode 不是"取最后一行"

Prefill: X [N,d] → 末层取末行 [1,d]。
Decode: X [1,d] 进 [1,d] 出，本就一行，不必"取"。
每步只送入一个新 token 的 embedding。

每个 token 都先过 Embedding

第 500 个生成 token 也得查表转向量。
token_id → Embedding[id] → [1,d]
前面 token 的信息全在 KV Cache 里。

Unembedding 层

末行 [1,d] × WvocabT [d,V] = Logits [1,V]
V = 词汇表大小（几万～几十万）。
通常与 Embedding 共享权重。
常见误解 vs 实际
误解：缓存命中 → 系统把缓存文本塞回 prompt → LLM 重算
实际：缓存命中 → 直接加载 K/V 矩阵到 GPU → 只算新增 token
